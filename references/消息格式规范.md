# 消息格式规范 — 飞书多Agent跨实例协作

> **版本**：v1.0 | 适用范围：所有参与协作的Bot

---

## 核心原则

**每条发给对方的消息，都必须在开头包含正确的`<at>`标签。**

---

## 唯一正确的@格式

```xml
<at user_id="ou_对方的open_id">对方名字</at> 消息内容
```

**⚠️ user_id必须是open_id（ou_开头），不是cli_会话ID！**

---

## 正确vs错误示范

### ✅ 正确

```xml
<at user_id="ou_47eb6e4cddfcebaa8f1150a16e88713b">珠珠</at> 请帮我查一下这个数据
<at user_id="ou_fc4fbfecb748e0093462334de71e669a">菲菲</at> 数据已查好，结果如下...
```

### ❌ 错误

```xml
<!-- 错误1：使用了cli_会话ID代替open_id -->
<at user_id="cli_a9637d13d4f95bb3">珠珠</at> 请帮我查一下
<!-- 原因：cli_是会话ID，gateway不会识别为用户标识 -->

<!-- 错误2：user_id为空 -->
<at user_id="">珠珠</at> 请帮我查一下
<!-- 原因：user_id留空，消息不会推送给珠珠 -->

<!-- 错误3：没有at标签 -->
珠珠请帮我查一下
<!-- 原因：没有at标签，gateway不知道这条消息要路由给谁 -->
```

---

## ID类型说明

| ID类型 | 格式 | 来源 | 用途 | 用于@标签？ |
|--------|------|------|------|------------|
| open_id | `ou_`开头 | 飞书分配 | 用户的唯一标识 | ✅ 是 |
| cli_会话ID | `cli_`开头 | Hermes分配 | Bot的会话标识 | ❌ 否 |
| 群ID | `oc_`开头 | 飞书分配 | 群的唯一标识 | 不用于@ |

---

## 私下会话路由（不经过群）

有时两个Bot需要私下通信，不经过飞书群。这时的格式不同：

| 发送方 | 目标Bot | 路由格式 |
|--------|---------|---------|
| 主持者Bot | 执行者Bot | `feishu:cli_执行者的cli_id` |
| 执行者Bot | 主持者Bot | `feishu:cli_主持者的cli_id` |

**⚠️ 这是私下路由用的，不是@标签！@标签里永远用open_id。**

## 消息结构模板

### 派发任务（主持者→执行者）

```
<at user_id="ou_执行者open_id">执行者名字</at> 
任务：xxxxx
要求：
1. xxx
2. xxx
完成后请@我汇报结果
```

### 汇报结果（执行者→主持者）

```
<at user_id="ou_主持者open_id">主持者名字</at> 
任务完成，结果如下：
[结果内容]
请审核。
```

### 确认收到（任方向）

```
<at user_id="ou_对方open_id">对方名字</at> 收到。
```

---

## 通信链路验证流程

1. **主持者发送测试消息**
   ```
   <at user_id="ou_执行者open_id">执行者名字</at> 通信测试，请回复"收到"
   ```

2. **执行者回复确认**
   ```
   <at user_id="ou_主持者open_id">主持者名字</at> 收到，通信正常
   ```

3. **若无回复** → 参见 `故障排查清单.md`
